## План (по пунктам, последовательно)

### Пункт 1 — Fix RLS bypass (Bug #1+#2)
**Файл:** `apps/api/app/modules/auth/service.py`
**Что:** В `create_user_and_tokens()` перед `db.flush()` вызвать
`SELECT set_config('app.tenant_id', :tid, true)` чтобы RLS policy
`tenant_isolation` пропустил INSERT нового user.
**Что проверить:** `/auth/register` → 200, `/auth/demo-login` → 200 для всех ролей.
**Статус:** ⏳

### Пункт 2 — Fix SQLAlchemy 2.0 crossjoin (Bug #3)
**Файл:** `apps/api/app/modules/admin/service.py:288` (внутри `get_activity_summary`)
**Что:** Заменить `.crossjoin(table)` на SQLAlchemy 2.0 syntax — `select(...).join(other, full=True)`.
**Что проверить:** `/admin/dashboard` → 200.
**Статус:** ⏳

### Пункт 3 — Lazy-create tenant_settings (Bug #5)
**Файл:** `apps/api/app/modules/tenants/` (router или service)
**Что:** При `GET /tenant-settings/me` (или какой там endpoint) если строки
нет — создавать дефолтную с разумными defaults.
**Что проверить:** для одного из tenants — row создан, GET возвращает 200.
**Статус:** ⏳
