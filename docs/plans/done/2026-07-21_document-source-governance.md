# Управление источниками AI-курса

Дата: 2026-07-21

Статус: реализовано, проверено и развёрнуто в production

Production revision: `5bc86c61591bc4bea849b7700a70ef7efd4a27ee`

## Цель

Исключить случайное смешивание документов из разных предметных областей и сделать происхождение AI-контента проверяемым на всём жизненном цикле курса: от выбора документов до ручного редактирования и утверждения методологом.

## Продуктовое поведение

1. После выбора документов система автоматически строит их semantic-профили и проверяет совместимость.
2. Однородный набор используется как единый источник курса.
3. Неоднородный набор разбивается на тематические группы. Генерация блокируется до решения методолога.
4. Методолог может:
   - выбрать одну тематическую группу и создать отдельный курс;
   - осознанно объединить группы, указав общую учебную цель длиной не менее 20 символов.
5. Backend повторяет проверку перед списанием trial-лимита и постановкой AI-задачи. Обход проверки через прямой API невозможен.
6. Architect обязан назначить каждому уроку документы-источники из выбранного набора.
7. Writer извлекает материал только из назначенных документов. Если релевантных фрагментов нет, генерация останавливается вместо подстановки общих знаний модели.
8. В курсе сохраняются исходные документы, стратегия объединения, общая цель и результат анализа совместимости.
9. В каждом уроке сохраняются фактически использованные документы и фрагменты. Они отображаются на экране проверки курса.
10. После ручного изменения заголовка или содержания урок получает статус `needs_review`. Повторная grounded-генерация возвращает `verified`; утверждение курса методологом также фиксирует явную проверку.
11. При недоступности Qwen используется Voyage. Если оба semantic-провайдера недоступны, индексация завершается понятной ошибкой; случайные hash-векторы не создаются.

## Реализация

### Анализ совместимости

- Tenant-scoped сервис: `apps/api/app/modules/ai/source_analysis.py`.
- API: `POST /api/v1/ai/document-compatibility`.
- Для каждого документа вычисляется центроид его chunk embeddings.
- Complete-link clustering не позволяет связать две далёкие темы через один промежуточный документ.
- Статусы результата: `compatible`, `mixed`, `incompatible`.
- Конфликт генерации возвращается как структурированный `409` с кодом `mixed_document_topics` и тематическими группами.

### Контракт генерации

- `source_strategy`: `single_topic` или `intentional_combination`.
- `combination_goal` обязателен при намеренном объединении.
- `target_audience`, стратегия и цель передаются в Architect.
- `source_doc_ids` уроков валидируются относительно выбранного набора.

### Трассируемость

Миграция `0068_course_source_governance.py` добавляет:

- к курсу: `source_document_ids`, `source_strategy`, `source_combination_goal`, `source_analysis`;
- к уроку: `source_document_ids`, `source_references`, `source_validation_status`;
- GIN-индексы по наборам документов и check constraints по статусам.

### UX

- Анализ запускается автоматически после выбора готовых документов.
- При конфликте показываются тематические группы, состав документов и связность группы.
- Доступны явные действия: использовать одну группу или объединить с общей целью.
- На review-экране отображаются источники курса и каждого урока.
- Уроки после ручных изменений помечаются как требующие проверки источников.
