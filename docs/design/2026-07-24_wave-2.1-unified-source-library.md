# Wave 2.1 — единая библиотека источников Kamilya LMS

Статус: **implementation design, production code не изменён**
Дата: 2026-07-24
Владелец продукта и операций: **methodologist**
Канонический экран: **`/documents`**
Зависимость: Gate 1 должен быть принят до начала реализации Wave 2.1.

## 1. Границы, термины и способ чтения

Документ проектирует единый каталог файлов, которые методолог использует как
источники для AI-курсов и как должностные инструкции. Он не проектирует заново
должности, компетенции, Assignment Hub или генератор курсов: для них описаны
только интерфейсные контракты.

Метки:

- **[CURRENT]** — подтверждённое поведение текущего кода; каждое такое
  утверждение сопровождается `file:line`.
- **[PROPOSED]** — обязательный или optional контракт Wave 2.1.
- **[DECISION]** — решение, которое нельзя безопасно принять исполнителю
  самостоятельно.

Под «источником» далее понимается существующая запись `documents`; отдельную
сущность или микросервис `sources` вводить не нужно.

## 2. Текущее состояние и подтверждённые дублирования

### 2.1. Один backend, два файловых менеджера

**[CURRENT]** `/documents` загружает весь tenant-список через
`GET /v1/documents`, проглатывает ошибку загрузки и не отличает её от пустого
результата (`apps/web/src/app/documents/page.tsx:43-49`). Страница имеет
собственное single-file modal, собственные title/description state и вызывает
`POST /v1/documents/upload` (`apps/web/src/app/documents/page.tsx:35-41`,
`apps/web/src/app/documents/page.tsx:72-92`). Она отображает только базовые
поля, не знает `category`, `embedding_status`, `embedding_error`, summary или
usages (`apps/web/src/app/documents/page.tsx:20-28`,
`apps/web/src/app/documents/page.tsx:200-220`).

**[CURRENT]** `/ai/generate` независимо загружает тот же полный список через
тот же endpoint (`apps/web/src/app/ai/generate/page.tsx:244-249`), но содержит
второй multi-file drag-and-drop uploader и последовательно вызывает тот же
`POST /v1/documents/upload` для каждого файла
(`apps/web/src/app/ai/generate/page.tsx:251-287`,
`apps/web/src/app/ai/generate/page.tsx:614-638`). Он также реализует второй
полный список документов с selection, статусами и delete проблемных записей
(`apps/web/src/app/ai/generate/page.tsx:646-728`).

**Подтверждённое дублирование:** upload transport, file picker/drop zone,
fetch/refresh списка, локальное отображение записей и delete присутствуют в
двух экранах. Это не две библиотеки данных, но два расходящихся UI-менеджера
одной таблицы.

### 2.2. Обычные документы и должностные инструкции

**[CURRENT]** `documents.category` допускает только `general` и
`job_instruction` (`apps/api/app/models/document.py:22-35`), а upload валидирует
тот же набор (`apps/api/app/modules/documents/router.py:183-193`).

**[CURRENT]** upload должностной инструкции не создаёт второе хранилище:
`POST /positions/{position_id}/instruction` анализирует файл, затем напрямую
переиспользует `upload_document(..., category="job_instruction")`
(`apps/api/app/modules/positions/jd_router.py:162-190`) и сохраняет
`positions.instruction_document_id` (`apps/api/app/modules/positions/jd_router.py:192-212`).
Связь имеет `ON DELETE SET NULL`
(`apps/api/app/modules/positions/models.py:85-101`).

**[CURRENT]** замена инструкции создаёт snapshot только текстовых полей
responsibilities/requirements в `position_jd_versions`, но не версию файла
(`apps/api/app/modules/positions/jd_router.py:192-203`,
`apps/api/app/modules/positions/models.py:124-141`). Повторная загрузка
`job_instruction` с тем же filename создаёт отдельную запись, тогда как
`general` возвращает существующую запись по `(tenant_id, filename, category)`
без сравнения содержимого (`apps/api/app/modules/documents/router.py:221-234`).

### 2.3. Индексация и повторное использование

**[CURRENT]** запись создаётся со статусом `pending`, после чего upload в том же
HTTP request синхронно запускает ingestion (`apps/api/app/modules/documents/router.py:252-272`).
Статусы модели: `pending | success | failed`
(`apps/api/app/modules/documents/schemas.py:18-20`). Частичная индексация
помечается `success`, а потеря chunks записывается строкой в `embedding_error`
(`apps/api/app/modules/documents/router.py:277-299`). Следовательно, текущий
`success` не всегда означает полное покрытие.

**[CURRENT]** wizard разрешает выбор только `success` и блокирует генерацию при
любом выбранном non-success (`apps/web/src/app/ai/generate/page.tsx:157-166`,
`apps/web/src/app/ai/generate/page.tsx:291-297`). Сервер повторно проверяет
tenant, наличие записей, `embedding_status == success` и наличие реальных
embedding rows (`apps/api/app/modules/ai/source_analysis.py:149-200`).

**[CURRENT]** один источник можно повторно выбрать для разных генераций.
`AIGenerateRequest.documents` принимает 1–20 UUID
(`apps/api/app/modules/ai/schemas.py:8-16`); выбранные UUID сохраняются в
`ai_jobs.params` (`apps/api/app/modules/ai/router.py:139-153`) и затем в
`courses.source_document_ids` и `lessons.source_document_ids`
(`apps/api/app/modules/ai/pipeline.py:80-104`,
`apps/api/app/modules/ai/pipeline.py:128-150`).

**[CURRENT]** для должностной инструкции имеются дополнительные usages:
`positions.instruction_document_id`, `courses.source_instruction_id`; оба FK
обнуляются при delete (`apps/api/alembic/versions/0063_job_instruction_sources.py:37-89`).
Обычные course/lesson source links — JSONB UUID arrays с GIN indexes, а не FK
(`apps/api/alembic/versions/0068_course_source_governance.py:18-90`).

### 2.4. Удаление

**[CURRENT]** `DELETE /documents/{id}` проверяет tenant и сразу вызывает
`db.delete(doc)`; проверок связанных position/course/lesson/job usages нет
(`apps/api/app/modules/documents/router.py:354-366`). Код handler также не
удаляет объект из storage и строки `document_embeddings`.

**[CURRENT]** поэтому FK-ссылки position/course-instruction обнуляются, но UUID
может остаться в JSONB source arrays и исторических `ai_jobs.params`. Это
создаёт «удалённый источник» в provenance. Отсутствие FK для JSONB подтверждено
моделью course (`apps/api/app/modules/courses/models.py:23-33`) и схемой
миграции (`apps/api/alembic/versions/0068_course_source_governance.py:18-53`).

## 3. Целевой UX

### 3.1. Информационная архитектура

**[PROPOSED, mandatory]** `/documents` становится единственным полным файловым
менеджером и называется «Библиотека источников».

Верхнеуровневые категории:

1. **Общие документы** → `category=general`;
2. **Должностные инструкции** → `category=job_instruction`.

Категория — фильтр/вкладка над одной коллекцией, не отдельная route и не
отдельная таблица. По умолчанию «Все источники».

У каждого source row/card:

- checkbox;
- title, filename, type, size;
- category;
- индексный статус: `processing`, `ready`, `partial`, `failed`;
- version marker и дата обновления;
- usages count с раскрытием;
- actions: скачать, посмотреть usages, повторить индексацию, удалить.

Поиск server-side по `title`, `filename`, `description`; фильтры: category,
index status, `used=true|false`, created date. Sort: newest (default), oldest,
title A–Я, size. Filter state живёт в URL query.

### 3.2. Upload и версии

**[PROPOSED, mandatory]** основной upload живёт на `/documents`. Он поддерживает
несколько файлов, но каждую запись показывает отдельно: upload → processing →
ready/partial/failed. Общая progress bar не скрывает частичные ошибки.

Для `general` совпадение filename больше не должно молча возвращать старую
запись. Сервер отвечает explicit duplicate conflict с existing source; UI
предлагает:

- использовать существующий;
- загрузить новую версию;
- отменить.

Новая версия — новая `documents` row, связанная с logical source family. Старые
курсы сохраняют старый document UUID; новая генерация по умолчанию выбирает
актуальную версию. Для `job_instruction` replacement остаётся инициируемым из
позиции, но созданный файл сразу виден в общей библиотеке.

### 3.3. Wizard генерации

**[PROPOSED, mandatory]** первый шаг `/ai/generate` — selector, а не второй
файловый менеджер:

- строка поиска;
- category/status chips;
- paginated список только selectable sources;
- выбранные источники закреплены над списком;
- «Открыть библиотеку» — link в `/documents`;
- compact «Добавить новый источник» раскрывает минимальный uploader.

Compact upload вызывает общий API/client component, создаёт запись в библиотеке
и автоматически добавляет её в selection только после `ready`. В wizard нет
delete, usages management, title/description editor и полного списка
непригодных источников. Ошибка предлагает «Повторить индексацию» или «Открыть
источник в библиотеке».

Compatibility и multi-domain decision сохраняются: текущий wizard уже запускает
debounced `POST /ai/document-compatibility`
(`apps/web/src/app/ai/generate/page.tsx:183-211`) и требует explicit learning
goal для intentional combination (`apps/web/src/app/ai/generate/page.tsx:731-811`).

### 3.4. Состояния

| Состояние | `/documents` | Wizard selector |
|---|---|---|
| Empty tenant | объяснение + primary upload | «Источников нет» + compact upload |
| Empty filter | сбросить фильтры | сбросить фильтры |
| Loading | skeleton rows, controls disabled | skeleton selector |
| List error | inline error + Retry; не показывать empty | inline error + Retry |
| Uploading | per-file progress | compact queue; остальной selector работает |
| Processing | badge + disabled retry/delete | виден, но disabled |
| Ready | selectable | selectable |
| Partial | warning + coverage + reindex | selectable с warning |
| Failed | error summary + retry | disabled + link в библиотеку |
| Usage load error | row сохраняется, usages «не удалось загрузить» | не применяется |
| Delete conflict | modal со списком usages | delete отсутствует |

## 4. API-контракт

Префикс ниже — `/v1`. Расширяется существующий documents router; новый сервис
не вводится.

### 4.1. Текущие endpoints и DTO

| Endpoint | Current request | Current response/semantics |
|---|---|---|
| `GET /documents` | нет query | `list[DocumentResponse]`, весь tenant, newest first (`apps/api/app/modules/documents/router.py:154-165`) |
| `GET /documents/{id}` | UUID | `DocumentResponse`, tenant-scoped 404 (`apps/api/app/modules/documents/router.py:168-180`) |
| `POST /documents/upload` | multipart `file,title,description,category` | synchronous storage+ingestion, `201 DocumentResponse` (`apps/api/app/modules/documents/router.py:183-323`) |
| `GET /documents/{id}/download` | UUID | bytes с attachment filename (`apps/api/app/modules/documents/router.py:326-350`) |
| `DELETE /documents/{id}` | UUID | `204`, без usage guard (`apps/api/app/modules/documents/router.py:354-366`) |
| `POST /ai/document-compatibility` | `{documents: UUID[1..20]}` | status/score/decision/clusters (`apps/api/app/modules/ai/schemas.py:25-46`) |
| `POST /ai/generate-course` | `AIGenerateRequest` | `202 AIJobResponse` (`apps/api/app/modules/ai/router.py:101-115`, `apps/api/app/modules/ai/schemas.py:49-56`) |

Текущий `DocumentResponse` содержит internal `tenant_id`, `uploaded_by`,
`s3_key`; публичная схема также содержит category/index status/error и summary
(`apps/api/app/modules/documents/schemas.py:8-26`). Wave 2.1 не должна отдавать
`s3_key`; tenant определяется auth context.

### 4.2. `GET /documents` — paginated catalog

**[PROPOSED, mandatory]**

Query:

```text
q: string <= 200
category: general | job_instruction
index_status: processing | ready | partial | failed
used: boolean
created_from, created_to: ISO-8601
sort: created_desc | created_asc | title_asc | size_desc
cursor: opaque string
limit: 1..100, default 25
include: usages_summary
```

Response:

```json
{
  "items": [{
    "id": "uuid",
    "title": "string",
    "filename": "string",
    "content_type": "string",
    "size": 123,
    "description": "string",
    "category": "general",
    "index": {
      "status": "ready",
      "error_code": null,
      "message": null,
      "chunks_total": 42,
      "chunks_indexed": 42,
      "indexed_at": "timestamp",
      "revision": 3
    },
    "version": 2,
    "is_latest": true,
    "created_at": "timestamp",
    "updated_at": "timestamp",
    "usages_summary": {"total": 4, "courses": 3, "positions": 1}
  }],
  "page": {"next_cursor": "opaque-or-null", "has_more": false, "limit": 25}
}
```

Cursor включает sort key + UUID и подписывается/валидируется сервером. Search и
фильтры применяются до cursor. `used` вычисляется по всем четырём current
domains: positions, instruction courses, course JSONB, lesson JSONB. AI job
history показывается в detail, но не удерживает operational delete само по себе.

### 4.3. Bulk selection

**[PROPOSED, mandatory]** UI selection across pages хранит явные UUID, не
«выбрать всё по фильтру». Генератор всё равно принимает максимум 20 UUID.

Добавить:

```http
POST /documents/resolve-selection
{"ids":["uuid", "..."], "require_index_ready":true, "max_items":20}
```

```json
{
  "items": [DocumentListItem],
  "rejected": [{"id":"uuid","code":"not_ready|not_found|wrong_tenant"}]
}
```

Cross-tenant UUID всегда маскируется как `not_found`; response не раскрывает
существование чужой записи. Это расширяет текущую server validation, которая
уже tenant-scoped и проверяет readiness/embeddings
(`apps/api/app/modules/ai/source_analysis.py:158-200`).

### 4.4. Usage detail

**[PROPOSED, mandatory]**

```http
GET /documents/{id}/usages?cursor=&limit=25
```

```json
{
  "document_id": "uuid",
  "summary": {"total":4,"positions":1,"courses":2,"lessons":1,"active_jobs":0},
  "items": [
    {
      "type": "position_instruction|course_instruction|course_source|lesson_source|active_ai_job",
      "id": "uuid-or-job-id",
      "title": "string",
      "status": "draft|published|running|...",
      "route": "/positions/{id}",
      "blocks_delete": true
    }
  ],
  "page": {"next_cursor": null, "has_more": false, "limit": 25}
}
```

Route строится из allowlist по type, не принимается из БД. Все joins включают
tenant predicate, даже при наличии RLS.

### 4.5. Protected delete

**[PROPOSED, mandatory]** `DELETE /documents/{id}`:

- `204` только если blocking usages = 0 и нет ingestion/reindex job;
- `404` для отсутствующего/чужого id;
- `409` при usages;
- `423` при processing/reindex lock;
- после commit/outbox cleanup удаляются embeddings, summary и blob; повторный
  cleanup идемпотентен.

Conflict:

```json
{
  "detail": {
    "code": "document_in_use",
    "message": "Источник используется и не может быть удалён",
    "document_id": "uuid",
    "blocking_usages": {
      "total": 3,
      "items": [
        {"type":"course_source","id":"uuid","title":"Курс","status":"published"}
      ],
      "truncated": false
    }
  }
}
```

Wave 2.1 не добавляет `force=true`: автоматическое вычищение provenance из
курсов опасно и выходит за scope. Сначала пользователь отвязывает источник в
каноническом owner-flow соответствующего объекта.

### 4.6. Retry/reindex и upload

**[PROPOSED, mandatory]**

```http
POST /documents/{id}/reindex
If-Match: "<index_revision>"
Idempotency-Key: "<uuid>"
```

Ответ `202`:

```json
{
  "job_id": "string",
  "document_id": "uuid",
  "status": "pending",
  "index_revision": 4
}
```

Использовать существующую `ai_jobs` инфраструктуру и tenant-scoped job helpers:
модель уже хранит status/stage/progress/params/errors
(`apps/api/app/models/ai_job.py:8-25`), а service создаёт и обновляет DB-backed
jobs (`apps/api/app/modules/ai/job_service.py:12-69`). В `params` добавить
`action=document_reindex, document_id, expected_revision`. Для одного
`(tenant_id, document_id)` допускается только один active reindex. Старый worker
не может перезаписать результат новой revision.

Upload сохраняет multipart contract, но:

- duplicate general filename/content возвращает `409 duplicate_document`;
- `create_new_version_of=<uuid>` создаёт новую version;
- HTTP request заканчивается после durable blob + document row; ingestion
  становится job, поэтому `201` возвращает `index.status=processing`.

**[PROPOSED, optional]** endpoint batch upload не нужен: браузер может вызвать
существующий endpoint параллельно с concurrency 2–3 и показать per-file result.

## 5. Data model impact и миграции

### 5.1. Переиспользовать

- `documents` — canonical source metadata/blob pointer
  (`apps/api/app/models/document.py:9-29`);
- `document_embeddings` — chunk index, tenant_id + doc_id
  (`apps/api/alembic/versions/0018_add_document_embeddings.py:20-36`);
- `positions.instruction_document_id`
  (`apps/api/app/modules/positions/models.py:85-101`);
- `courses.source_instruction_id`, `courses.source_document_ids`
  (`apps/api/app/modules/courses/models.py:23-33`);
- `lessons.source_document_ids` и source references
  (`apps/api/alembic/versions/0068_course_source_governance.py:47-77`);
- `ai_jobs` для reindex orchestration.

### 5.2. Обязательная миграция

Нужна одна reversible migration:

```text
documents:
  source_family_id UUID NOT NULL
  version INTEGER NOT NULL DEFAULT 1
  content_sha256 CHAR(64) NULL initially, then NOT NULL after backfill
  index_status TEXT NOT NULL DEFAULT 'processing'
  index_error_code TEXT NULL
  index_message TEXT NULL
  index_chunks_total INTEGER NULL
  index_chunks_indexed INTEGER NULL
  index_revision INTEGER NOT NULL DEFAULT 1
  indexed_at TIMESTAMPTZ NULL

indexes:
  (tenant_id, category, created_at DESC, id)
  (tenant_id, source_family_id, version DESC)
  unique (tenant_id, source_family_id, version)
  (tenant_id, content_sha256)
```

Backfill: `source_family_id=id`, `version=1`; map `pending→processing`,
`failed→failed`; map `success + embedding_error starts "Partial:"→partial`,
остальные `success→ready`. Chunks остаются NULL, пока reindex не даст точные
значения. `embedding_status/error` остаются на один compatibility release, затем
удаляются отдельной будущей migration.

`document_embeddings.doc_id` сейчас `TEXT`, поэтому FK cascade добавлять в этой
волне рискованно (`apps/api/alembic/versions/0018_add_document_embeddings.py:22-30`).
Cleanup выполняет documents service в транзакционно-идемпотентном workflow.

Rollback: удалить новые indexes/columns; перед downgrade синхронизировать
`embedding_status/error` из новых полей. Новые version rows остаются обычными
documents rows и не теряются.

### 5.3. Не делать в Wave 2.1

- не нормализовать JSONB course/lesson provenance в новую join table;
- не переносить position JD snapshots в source versions;
- не создавать source microservice;
- не менять position/competency/assignment ownership;
- не изменять delivery/publication semantics курсов.

## 6. Permission и tenant isolation

Целевой продуктовый владелец — methodologist. Текущий router допускает
`superadmin` и `methodologist` (`apps/api/app/modules/documents/router.py:154-190`);
roles guard возвращает 403 для остальных (`apps/api/app/core/auth.py:293-306`).
Tenant context обязателен (`apps/api/app/modules/documents/router.py:18-22`,
`apps/api/app/core/auth.py:317-341`).

| Actor | List/get/download | Upload/reindex | Usages | Delete |
|---|---:|---:|---:|---:|
| methodologist, свой tenant | allow | allow | allow | allow, если нет conflict |
| admin / org_admin | deny 403 | deny 403 | deny 403 | deny 403 |
| student | deny 403 | deny 403 | deny 403 | deny 403 |
| superadmin без tenant context | deny 403 | deny 403 | deny 403 | deny 403 |
| superadmin с explicit impersonated tenant context | allow для support, audit required | allow, audit required | allow | allow, audit required |
| любой actor, чужой UUID | 404 | 404 | 404 | 404 |

Invariants:

1. Каждый ORM/SQL query содержит `tenant_id=user.tenant_id`.
2. Storage key сохраняет tenant prefix; текущий upload уже формирует
   `tenants/{tenant_id}/documents/{doc_id}` (`apps/api/app/modules/documents/router.py:236-241`).
3. `documents` и `document_embeddings` входят в tenant RLS policy
   (`apps/api/alembic/versions/0042_rls_empty_tenant_context_safe.py:17-46`) и
   FORCE RLS set (`apps/api/alembic/versions/0033_force_rls_and_lms_app_role.py:54-90`).
4. Empty tenant setting fail-closed через `NULLIF(..., '')`
   (`apps/api/alembic/versions/0042_rls_empty_tenant_context_safe.py:61-68`).
5. Cross-tenant API остаётся indistinguishable 404; существующие tests уже

## 8. Gate 2.1

Gate проходит только если одновременно:

- `/documents` — единственный полный manager; wizard использует selector +
  compact upload;
- general и job instruction видны и фильтруются в одной tenant library;
- list server-paginated, search/filter/cursor детерминированы;
- ready/partial/failed отражают реальные chunk counts;
- usages охватывают position, course instruction, course/lesson provenance и
  active job;
- связанный источник нельзя удалить, а 409 объясняет, где он используется;
- reindex идемпотентен и concurrent jobs не перезаписывают новую revision;
- duplicate upload не возвращает молча старый документ;
- admin/org_admin/student получают 403, cross-tenant UUID — 404;
- RLS/FORCE и explicit tenant predicates проверены тестами;
- migration проходит upgrade/downgrade на clean и populated DB;
- desktop/tablet/mobile states проверены без horizontal page scroll;
- существующие compatibility/generation/JD flows не регрессировали.

## 9. Риски и контрмеры

| Риск | Текущая причина | Обязательная контрмера |
|---|---|---|
| Удаление связанного документа | delete не проверяет usages (`apps/api/app/modules/documents/router.py:354-366`) | 409 + usage detail; no force delete |
| Stale embeddings | status хранится отдельно от фактических rows | revision + chunk counts + stale worker guard |
| Multiple domains | current compatibility уже требует решение при mixed topics (`apps/api/app/modules/ai/router.py:115-125`) | сохранить compatibility и explicit goal |
| Job concurrency | AI job status DB-backed, но source lock отсутствует | unique active reindex per source + idempotency key |
| Duplicate uploads | general dedupe только по filename и молча возвращает старое (`apps/api/app/modules/documents/router.py:221-234`) | SHA-256 + explicit conflict/version choice |
| Language variants | language задаётся генерации, не source (`apps/api/app/modules/ai/schemas.py:10-16`) | не считать перевод новой версией автоматически; отдельная family decision |
| Large files | API limit 10 MB (`apps/api/app/modules/documents/router.py:41-44`, `apps/api/app/modules/documents/router.py:199-204`) | показать limit до upload; server remains authority |
| Partial index failure | partial сейчас маскируется как success (`apps/api/app/modules/documents/router.py:288-296`) | отдельный `partial`, exact counts, reindex |
| Blob/DB split-brain | blob записывается до DB row (`apps/api/app/modules/documents/router.py:236-266`) | idempotent cleanup/reconciliation job |
| Provenance JSONB без FK | source UUID arrays не cascade | usage guard; не force-delete |

## 11. Нерешённые решения

1. **[DECISION] Language family:** русский/казахский/английский перевод — новая
   version одной family или независимый source family? Рекомендация: независимая
   family до появления подтверждённого translation provenance.
2. **[DECISION] Published provenance retention:** сколько хранить blob после
   архивирования всех usages? Рекомендация: soft archive/retention policy
   отдельной волной; Wave 2.1 разрешает hard delete только unused.
3. **[DECISION] Partial threshold:** любой `0 < indexed < total` = partial или
   допустим процент потерь? Рекомендация: всегда partial; selectable с warning.
4. **[DECISION] Superadmin support:** разрешить ли impersonated write/delete или
   оставить read-only? Матрица выше допускает write только при audit, но
   окончательная политика требует product/security approval.
5. **[DECISION] Version replacement UX для job instruction:** выбор существующей
   library instruction в position flow или только upload нового файла?
   Рекомендация: оба варианта, но реализация selector — interface contract с
   Wave 2.2, не расширение этого backlog.
