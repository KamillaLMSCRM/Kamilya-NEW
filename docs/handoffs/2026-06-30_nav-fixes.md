## Что было сломано

UI-баг: методолог/админ нажимает «Привязать» в `/admin/staff?tab=rules`,
получает 422 Unprocessable Entity. Курс не привязывается.

Корень — три бага и один архитектурный косяк:

1. **Frontend** (`apps/web/src/app/admin/staff/_tabs/RulesTab.tsx:97-102`):
   `id: d.id ?? d.slug`. `d.id` для Excel-импортированных тенантов
   = null (outer-join в `/v1/admin/staff/structure` возвращает null).
   Fallback на slug (Title case, как «HR», «IT», «Маркетинг»).
2. **Backend** (`apps/api/app/modules/departments/router.py`): принимал
   `department_id: UUID`. Slug → 422.
3. **Schema** (`apps/api/app/models/department.py`): ORM не объявлял
   колонки `description`, `code`, `head_user_id`, `created_at`, которые
   давно были в БД. `description: nullable=True` (в ORM) vs `NOT NULL`
   (в БД). При auto-create — `NotNullViolationError` на каждом attach.
4. **Pre-existing dormant bug** (`apps/api/app/modules/positions/batch_service.py:83, 105, 126`):
   `[row[0] for row in result.scalars().all()]`. `scalars()` для
   single-column SELECT возвращает values напрямую, не Row; `row[0]`
   пытается `asyncpg.UUID[0]` → `TypeError: not subscriptable`. Спал
   пока `position_ids` был пустым (legacy Excel без `department_id`).
5. **UI-дубли:** сайдбар имел «Штатка», «Привязка курсов»,
   «Курсы компании» — все вели на `/admin/staff?tab=...` (одна страница).
   Плюс таб внутри назывался «Правила» (не «Привязка курсов»).
